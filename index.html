Askboard

LLM-powered Q&A over hotel/restaurant billing documents

NDA — description only

Problem

Hotel and restaurant bills are unstructured and inconsistent in format, making it hard to quickly extract or query specific details — charges, taxes, discounts — without manual review.

What I built

A web app that lets users upload a bill, extracts its content via OCR, and answers natural language questions about it using an LLM grounded in the extracted data.

My role

Built the OCR pipeline, LLM query logic (including model routing for cost efficiency), and the frontend UI for client interaction.

Stack

Google Vision API (OCR) DeepSeek-V4-Flash DeepSeek-V4-Pro JavaScript Rate-limited spend: $10

Architecture

Bill image → Google Vision OCR → structured text extraction → indexed as a query-able knowledge base → user question routed to a mixed model pipeline → answer returned grounded in the extracted bill content.

V4-Flash · 90%
Flash — bulk of queries Pro — harder queries

Challenge solved

Designed a cost-efficient model routing strategy (90% Flash / 10% Pro) to balance answer quality against LLM API cost, instead of sending every query to the more expensive model.

Validation

Manually validated against 20+ real bill queries — model answers checked against manually verified correct answers.

Instant Itinerary Engine

Automated multi-source itinerary + PDF generator for a travel agency

NDA — description only

Problem

Travel agency itineraries required manually checking hotel rates across 4 disconnected sources — yearly Excel contracts, two login-only B2B portals, and public OTAs for price verification — plus transport costs, taking hours per client enquiry instead of the target 10 minutes.

What I built

An end-to-end pipeline that takes a single enquiry (destination, dates, pax, budget, preferences) and outputs 5 distinct itinerary PDFs — in the agency's own template — each with verified-cheapest hotel + transport pricing, within 10 minutes.

My role

Built the entire project from scratch — scraping infrastructure, job orchestration, itinerary engine, and PDF rendering.

Stack

Playwright / Puppeteer BullMQ Redis OpenRouter (LLM) Fal API (images) Excel sync layer

Architecture

Excel rate sheets normalized into a queryable DB, re-synced on update. On enquiry, parallelized scraping jobs (via BullMQ workers) pull live rates from B2B portals, cached 6–12 hrs so repeat destination searches are near-instant. OTA rates are scraped in parallel to confirm the agency's B2B rate is the cheaper option, flagging any mismatch. An itinerary engine selects hotel combinations across 5 price bands (budget → luxury) and merges in transport costs plus markup. 4 pricing variants per set (2 at actual cost, 2 with client markup) are generated via OpenRouter and rendered into branded PDFs using the agency's historical itinerary format as a design reference.

Challenge solved

Built resilient, parallelized browser automation across multiple login-gated portals with no API access, handling anti-scraping defenses like IP-based rate limiting — while keeping the full enquiry-to-PDF pipeline under a 10-minute SLA.

Validation

Average pipeline runtime of 10 minutes across 10 test enquiries, with agency rates verified cheaper than OTA pricing by 15%.
