Instant Itinerary Engine

Automated multi-source itinerary + PDF generator for a travel agency

NDA — description only

Problem

Travel agency itineraries required manually checking hotel rates across 4 disconnected sources — yearly Excel contracts, two login-only B2B portals, and public OTAs for price verification — plus transport costs, taking hours per client enquiry instead of the target 10 minutes.

What I built

An end-to-end pipeline that takes a single enquiry (destination, dates, pax, budget, preferences) and outputs 5 distinct itinerary PDFs — in the agency's own template — each with verified-cheapest hotel + transport pricing, within 10 minutes.

My role

Built the entire project from scratch — scraping infrastructure, job orchestration, itinerary engine, and PDF rendering.

Stack

Playwright / Puppeteer BullMQ Redis OpenRouter (LLM) Fal API (images) Excel sync layer

Architecture

Excel rate sheets normalized into a queryable DB, re-synced on update. On enquiry, parallelized scraping jobs (via BullMQ workers) pull live rates from B2B portals, cached 6–12 hrs so repeat destination searches are near-instant. OTA rates are scraped in parallel to confirm the agency's B2B rate is the cheaper option, flagging any mismatch. An itinerary engine selects hotel combinations across 5 price bands (budget → luxury) and merges in transport costs plus markup. 4 pricing variants per set (2 at actual cost, 2 with client markup) are generated via OpenRouter and rendered into branded PDFs using the agency's historical itinerary format as a design reference.

Challenge solved

Built resilient, parallelized browser automation across multiple login-gated portals with no API access, handling anti-scraping defenses like IP-based rate limiting — while keeping the full enquiry-to-PDF pipeline under a 10-minute SLA.

Validation

Average pipeline runtime of 10 minutes across 10 test enquiries, with agency rates verified cheaper than OTA pricing by 15%.

Voice Agent

Custom-built conversational voice AI — built from scratch, not on a platform

NDA — description only

Problem

Existing no-code voice agent platforms (Bolna, Smallest.ai) only allow script/config-level changes — they suffer from high latency, hallucination, no reliable barge-in handling, weak objection handling, and repetitive responses, with no way to fix any of it at the architecture level.

What I built

A voice agent built entirely from scratch (not on top of any existing platform) — covering telephony, VAD, STT, LLM reasoning, and TTS as a fully custom pipeline, giving full control over latency, conversation flow, and behavior instead of being limited to script tweaks.

My role

Designed and built the entire agent solo, end-to-end, in 8-9 days — telephony integration, VAD, latency optimization across the full pipeline, barge-in handling, and objection handling logic.

Stack

Plivo (telephony) AWS Lightsail n8n (automation/triggering)

Performance

End-to-end latency: 700–900 ms, including VAD and endpoint-hit latency.

STT: 250–300 ms LLM: 350–400 ms TTS: 100–200 ms

Languages: English + Hindi (Hinglish, Devanagari-aware). Handles real-time barge-in and mid-sentence interruption in both languages.

Cost: ₹3.50 per 3-min inbound call (~₹1.15/min), ₹4.50 per 3-min outbound call (~₹1.50/min).

Challenge solved

Built a full custom voice pipeline from the ground up in under 9 days, hitting sub-second end-to-end latency while solving the specific failure modes — hallucination, no barge-in, weak objection handling — that off-the-shelf platforms couldn't fix, without the luxury of extended fine-tuning time.

Use cases tested

Follow-up lead calls, client demo calls, and scheduled/automated call runs triggered via n8n.

Scheduling UI

Built a UI to schedule outreach across channels — voice calls through the agent, SMS via MSG91, WhatsApp via Chatwoot, and email via Brevo — from a single interface.
